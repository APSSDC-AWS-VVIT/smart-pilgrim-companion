## Presentation Narrative

1. Start with identity and context: Smart Pilgrim Companion is not just a static site; it is a cloud-ready full-stack platform.
2. Explain the problem: pilgrims need temple, route, budget, schedule, and planning information in one place.
3. Show the solution: React frontend, Flask APIs, planner/recommendation flow, deployment journey.
4. Prove the implementation: architecture, screenshots, APIs, deployment evidence, AWS monitoring.
5. Close with contribution, results, learning outcomes, and future scope.

## Minimal Slide Count Rationale

The viva deck is intentionally limited to 11 slides. The structure avoids a long report-style presentation and prioritizes:

- One idea per slide.
- Large visuals.
- Short speaker-friendly text.
- Proof from real screenshots.
- Strong architecture and deployment clarity.

## Core Message

Smart Pilgrim Companion brings temple discovery, trip planning, recommendation support, and cloud deployment together in one project. The final implementation demonstrates React + Vite, Flask, SQLAlchemy, AWS EC2, Nginx, Gunicorn, AWS RDS MySQL, IAM, CloudWatch, and GitHub Pages deployment.
